## motionAndFx

Three-category motion budget; everything else removed/gated. (1) State feedback ≤180ms: card hover bg/border, focus ring, button press (--motion-instant/quick). (2) Data-entry confirmation ≤500ms, fires ONCE on mount — this is the terminal's signature 'new data arrived' language: fade-up stagger, stat-bar-fill, chart-draw, count-up, duelGrow. (3) Live pulse — the ONLY allowed infinite animation, livePulse/.live-dot on genuinely in-progress games tied to real LiveState.
KEEP: scanline texture (static, near-free — but move to z-base behind content), dither header (page banners ONLY), dot-grid, count-up, decrypted-text, infield-divider clay accent, diamond watermark, scoreboard numerals.
CUT/REPLACE: rainbow ScoreTicker line (tickerLineSlide infinite gradient — competes with live-dot for 'live' meaning) → static --border line + green live-dot. Simultaneous glare + spotlight + shiny-text on the same card → pick ONE (subtle --surface-2 bg shift on data cards; reserve glare for the single POTD hero only). three.js/OGL liquid-chrome-bg on non-hero surfaces → already moving to CSS grain; keep dither only in banners.
DEDUPE: two conflicting @keyframes pulse (globals.css ~210 and ~543, later silently wins) → one pulse keyframe, one name; .live-dot references it.
REDUCED-MOTION: blanket @media (prefers-reduced-motion: reduce){ *,*::before,*::after{ animation:none!important; transition:none!important } } BUT preserve final states (bars at --fill, fades at opacity:1). Currently only 4 of 13 keyframes are guarded — this freezes scoreTicker/tickerLineSlide/livePulse/acePulse/duelGrow/glare/spotlight too.
PERF BUDGET: globally-mounted DotGrid (per-frame mouse canvas), NoiseOverlay (~50ms redraw), DitherHeader (three.js/ogl) gated behind prefers-reduced-motion AND navigator.connection.saveData AND coarse-pointer; pause all canvases on visibilitychange (tab hidden). Cap card stagger total at ~300ms (drop the index*35ms multiplier past ~12 cards) so a long mobile slate's bottom doesn't wait ~½s. Retire dead visual-tokens.ts dither constant or route DotGrid/dither colors through the typed token module.

## risks

- api.ts single-ownership: spec says it's READ-ONLY, but picks needs to SEED trackedIds from api.trackerBets on load. That uses the EXISTING endpoint (no api.ts edit) — implementers must not be tempted to add a helper to api.ts; keep the change inside picks/page.tsx (bucket B5).

- quant.tsx / explain.tsx / count-up.tsx are imported by picks, verify, track-record AND tools — if assigned to B5 they become a hidden shared dependency that B6/B7 edit too. Mitigation: split a B5a micro-bucket owning ONLY quant.tsx/explain.tsx/count-up.tsx that B5/B6/B7 all import read-only; do not let page buckets edit them.

- game-detail-panel.tsx is shared by slate drawer and /game/[id] — both placed in B4 so they move together; a parallel agent must NOT touch it from B5/B4 simultaneously.

- Border-collision split (--border==--surface-3) and demoting --text-3 are token VALUE changes, not just additions — they can shift the look of pages not yet migrated. Land them in B0 but visually QA every page after B0 before page buckets start.

- Removing render-blocking @import + retiring rainbow ticker touches the global shell (B2) — a regression there affects every page; gate behind a quick smoke test of all 11 routes.

- Reduced-motion blanket animation:none!important can freeze bars/fades at 0 if final-state preservation isn't implemented per-keyframe — must verify bars rest at --fill and fades at opacity:1.

- 60s get() revalidate cache vs 60s page polling on slate/live: redesign must not assume fresh live odds; verify the cache/poll interaction doesn't show stale 'updated Xs ago'.

- Tailwind v4 + inline-style + CSS-var hybrid: moving layout to utilities risks specificity fights with the existing !important media block — retire that block in the same bucket that introduces utilities, per page, not piecemeal.

- Mobile DataTable replacing the ::before data-label hack on tracker: every cell currently depends on correct data-label; the new component must cover ALL columns incl CLV chip + admin actions or mobile loses data.

- correlation_warning verbatim rule: any componentization of the tools result must pass the backend string through untouched — easy to accidentally reformat when building the new headline-first layout.
